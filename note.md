# CRUD 연습 프로젝트 — 학습 노트

할 일 목록(To-do) 앱을 만들며 웹 서비스 개발의 전체 흐름을 익히는 연습 프로젝트.
최종 목표는 이 감을 바탕으로 의료데이터 서비스를 직접 만드는 것.
(이 저장소에는 실제 환자 데이터·민감정보를 절대 올리지 않는다. 공개 데이터만 사용.)

## 큰 그림: 웹 서비스의 3칸
- 프론트엔드 — 브라우저·앱 화면 (사용자가 보는 것)
- 백엔드 — 로직·API (서버에서 도는 처리)
- 데이터베이스 — 데이터 저장소
- 요청→응답 루프: 프론트가 요청 → 백엔드가 처리 → 응답 → 화면에 표시

기술 선택: 백엔드=Flask, DB=SQLite, 프론트=순수 HTML/CSS/JS.
(각각 가장 단순·투명해서 초보가 흐름을 눈으로 볼 수 있는 도구)

## Step 1 — 첫 백엔드 서버 (완료)
목표: 백엔드 칸에 처음 불 켜기.

한 작업:
1. 가상환경: `python -m venv .venv` → `source .venv/bin/activate` → `pip install flask`
   - 프로젝트 전용 격리된 방. 새 터미널/세션마다 activate 다시 필요.
2. `app.py` 작성 (핵심 4줄)
   - `app = Flask(__name__)` : 서버 본체 생성 (Flask=라이브러리 부품, app=내가 지은 이름)
   - `@app.route("/")` : `/` 주소와 바로 아래 함수를 묶어 등록
   - `def home(): return "..."` : 그 요청에 대한 응답 내용
   - `app.run(debug=True)` : 서버 시동 (debug=코드 수정 시 자동 재시작)
3. 실행: `python app.py` → Codespaces PORTS 탭에서 접속 확인

배운 개념:
- home()을 내가 직접 부르지 않는다. 요청이 올 때 Flask가 대신 부른다 (제어의 역전).
- `@`는 아래 함수를 위 도구에 넘기는 문법 (데코레이터).
- return 문자열을 바꾸니 화면도 바뀜 = 내가 서버를 통제한다는 확인.
4. Git 저장: `.gitignore`(.venv/, __pycache__/ 무시) → `git add .` → `git commit` → `git push`
   - 흐름: 작업폴더 →(add) 후보 담기 →(commit) 역사에 박제 →(push) GitHub 백업
